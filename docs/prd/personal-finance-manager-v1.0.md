## 2. 需求背景与目标
### 2.1 需求背景
当前个人用户普遍存在以下财务管理痛点：
- 手动记账效率低，容易遗漏收支记录
- 消费去向不透明，无法清晰掌握每月开支结构
- 缺乏预算管控，经常出现超支情况
- 多平台资产分散，无法统一查看总资产情况
- 缺少消费趋势分析，无法制定合理的财务规划

### 2.2 项目目标
MVP版本核心目标：为用户提供轻量化、易用的个人财务管理工具，实现收支记录、预算管控、资产统计、消费分析四大核心能力，帮助用户清晰掌握财务状况，养成良好的理财习惯。

## 3. 目标用户
- 18-35岁的年轻白领、学生群体
- 有记账和财务管理需求的普通个人用户
- 非专业理财背景的普通消费者

## 4. 核心价值
✅ 收支快速记录：10秒内完成一笔收支记录，操作极简
✅ 消费透明可视：自动分类统计消费去向，一目了然
✅ 预算智能提醒：设置月度预算，超支实时提醒，避免冲动消费
✅ 资产统一管理：支持多账户资产录入，总资产情况一键查看
✅ 消费趋势分析：生成月度/季度消费报告，助力合理财务规划

## 6. 详细功能说明
### 6.1 收支记录模块
- 分类预设：支出分类包含餐饮、交通、购物、娱乐、医疗、教育、住房、通讯、人情、运动、其他；收入分类包含工资、奖金、投资收益、红包、其他
- 记录字段：类型（收入/支出）、分类、金额（必填）、时间（默认当前时间，可修改）、备注（可选）
- 列表展示：按时间倒序排列，显示分类图标、金额、备注、时间信息

### 6.2 预算管控模块
- 支持设置月度总预算，以及各分类的月度预算
- 预算进度展示：已使用金额/总预算，百分比进度条
- 提醒规则：当预算使用达到80%时，首页显示橙色提醒；达到100%时，显示红色提醒，新增对应分类支出时弹窗提示

### 6.3 资产统计模块
- 支持自定义添加账户，设置账户名称、类型、初始余额
- 总资产自动计算所有账户余额总和，展示在首页顶部
- 支持随时更新账户余额，余额变动自动关联收支记录（可选）

### 6.4 消费分析模块
- 趋势图：近6个月收支对比折线图
- 占比图：当月支出分类占比饼图，点击可查看分类详情
- 月度报告：包含总收支、结余、最大支出项、消费建议等内容

## 7. 用户操作流程
### 核心流程1：新增收支记录
1. 首页点击「+记一笔」按钮
2. 选择收入/支出类型
3. 选择对应分类
4. 输入金额
5. 可选填写备注、修改时间
6. 点击保存，记录成功，返回首页

### 核心流程2：设置预算
1. 进入「预算」页面
2. 点击设置月度总预算
3. 可选设置各分类预算
4. 保存设置，预算生效

### 核心流程3：查看资产
1. 进入「资产」页面
2. 查看总资产和各账户余额
3. 点击账户可修改余额或删除账户

### 核心流程4：查看消费分析
1. 进入「分析」页面
2. 查看收支趋势和分类占比
3. 下拉查看月度消费报告

## 8. 验收标准
### 通用验收标准
- 所有页面响应式适配，支持移动端和桌面端访问
- 页面加载速度≤2s
- 操作无卡顿，点击响应时间≤300ms
- 数据持久化可靠，刷新页面数据不丢失

### 功能验收标准
1. 收支记录模块
   - 新增记录必填项为空时，提示对应错误信息，无法保存
   - 记录保存后立即在列表中展示，数据准确
   - 编辑/删除记录后，列表数据同步更新
   - 按时间/分类筛选结果准确
2. 预算管控模块
   - 预算设置保存后，进度计算准确
   - 达到80%/100%预算时，提醒正确展示
   - 超支时新增对应分类支出，弹窗提示正确
3. 资产统计模块
   - 新增账户后，总资产计算准确
   - 修改账户余额后，总资产同步更新
   - 删除账户后，总资产同步扣除对应金额
4. 消费分析模块
   - 图表数据与实际收支数据一致
   - 月度报告数据准确，建议合理
   - 导出JSON文件格式正确，导入后数据完整

## 9. 非功能需求
- 兼容性：支持Chrome、Edge、Safari主流浏览器最新2个版本，支持微信内置浏览器
- 数据安全：本地数据加密存储，不会上传到第三方服务器
- 易用性：核心操作路径不超过3步，新用户无需教程即可上手
- 性能：单页面DOM节点≤1500个，内存占用≤100MB
