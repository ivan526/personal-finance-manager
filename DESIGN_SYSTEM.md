# FinSpace 设计系统 v1.0
现代简约风 UI/UX 规范 - 所有开发必须严格遵循

---

## 🎨 一、核心设计理念
**「极简专业、轻量化、高信息密度、无冗余装饰」**
- 功能优先，去除不必要的渐变、复杂阴影和装饰性元素
- 大面积留白，增强呼吸感，突出核心数据
- 冷静克制的色彩体系，传递金融产品的专业感和信任感
- 极致清晰的信息层级，用户3秒内可以获取页面核心信息

---

## 二、色彩系统（语义化Token）
### 基础色板
| 分类 | 色值 | 用途 | 说明 |
|------|------|------|------|
| 主色调 | `#165DFF` | 品牌色、主按钮、选中态、关键操作 | 蓝色系传递专业、可信，符合金融产品调性 |
| 成功色 | `#00B42A` | 收益、盈利、正增长、操作成功 | 低饱和度绿色，醒目但不刺眼 |
| 警告色 | `#FF7D00` | 预警、接近预算、中等风险 | 温和的橙色，避免过度焦虑 |
| 危险色 | `#F53F3F` | 亏损、超支、高风险、删除操作 | 低饱和度红色，避免视觉冲击过大 |
| 中性色-背景 | `#FFFFFF` / `#F2F3F5` | 页面背景、卡片背景 | 纯白+浅灰背景，清晰分离层级 |
| 中性色-文本 | `#1D2129` / `#4E5969` / `#86909C` / `#C9CDD4` | 标题/正文/辅助文字/禁用文字 | 四级灰度文本，清晰的层级对比 |
| 中性色-边框 | `#E5E6EB` / `#F2F3F5` | 边框、分割线 | 极浅灰色，弱分割，不干扰内容 |

### 暗模式配套（可选）
| 分类 | 色值 | 用途 |
|------|------|------|
| 背景 | `#121212` / `#1E1E1E` | 页面背景、卡片背景 |
| 文本 | `#F2F3F5` / `#C9CDD4` / `#86909C` | 标题/正文/辅助文字 |
| 边框 | `#2E2E2E` / `#3A3A3A` | 边框、分割线 |

### 使用规则
- 禁止直接使用十六进制色值，必须通过CSS变量引用
- 语义化使用颜色，禁止随意换色
- 颜色对比度必须满足WCAG 4.5:1标准

## 三、字体系统
### 字体栈
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;
```
采用系统无衬线字体，清晰易读，性能最优。

### 字体层级
| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| 大标题 | 32px | 700 | 1.3 | 页面核心指标（总资产、总收益等） |
| 标题 | 24px | 600 | 1.4 | 页面标题、卡片标题 |
| 副标题 | 18px | 600 | 1.4 | 次级标题、模块标题 |
| 正文 | 16px | 400/500 | 1.5 | 主要内容、表格正文 |
| 辅助文字 | 14px | 400 | 1.5 | 辅助说明、标签文字 |
| 小字 | 12px | 400 | 1.5 | 时间、提示文字、次要说明 |

### 特殊规则
- 所有金额、百分比数字使用等宽字体：`font-variant-numeric: tabular-nums`
- 禁止使用小于12px的文字
- 数字右对齐，文本左对齐，标题居中对齐

## 四、间距与布局系统
采用8px基础网格系统，所有间距必须是8px的倍数。

### 间距规范
| 类型 | 数值 | 用途 |
|------|------|------|
| 页面边距 | 24px（桌面）/ 16px（移动） | 页面左右内边距 |
| 模块间距 | 24px / 32px | 大模块之间的间距 |
| 组件间距 | 12px / 16px / 20px | 组件内部和组件之间的间距 |
| 内边距 | 12px / 16px / 20px / 24px | 卡片、按钮、输入框内边距 |

### 圆角规范
| 类型 | 数值 | 用途 |
|------|------|------|
| 小圆角 | 8px | 按钮、输入框、小卡片 |
| 中圆角 | 12px | 普通卡片、弹窗内容 |
| 大圆角 | 16px | 大容器、弹窗、模态框 |

### 阴影规范
克制使用阴影，仅保留两级：
| 层级 | 数值 | 用途 |
|------|------|------|
| 卡片阴影 | `0 2px 8px rgba(0, 0, 0, 0.06)` | 普通卡片、组件悬浮 |
| 弹窗阴影 | `0 8px 24px rgba(0, 0, 0, 0.12)` | 弹窗、下拉菜单、悬浮层 |

### 布局规则
- 内容最大宽度：1200px，居中对齐
- 响应式断点：768px（平板）、1024px（桌面）
- 移动端优先设计，布局自适应不同屏幕尺寸
- 禁止出现横向滚动条

## 五、组件规范
### 按钮
| 类型 | 样式 | 状态 |
|------|------|------|
| 主按钮 | 实心`#165DFF`背景，白色文字，圆角8px，高度40px/44px，字重600 | hover: 背景`#0D47A1`; active: 缩放0.98倍; disabled: 透明度0.4 |
| 次按钮 | 白色背景，`#E5E6EB`边框，灰色文字，圆角8px，高度40px/44px | hover: 边框`#165DFF`，文字`#165DFF`; active: 缩放0.98倍 |
| 文字按钮 | 透明背景，`#165DFF`文字，无框 | hover: 背景`rgba(22, 93, 255, 0.1)` |

### 卡片
- 白色背景，圆角12px，卡片阴影，无边框
- 内边距：20px / 24px
- hover效果：阴影加深为`0 4px 12px rgba(0, 0, 0, 0.08)`，上移1px，过渡200ms

### 输入框
- 白色背景，`#E5E6EB`边框，圆角8px，高度40px/44px，内边距左右16px
- focus状态：边框`#165DFF`，外发光`0 0 0 2px rgba(22, 93, 255, 0.2)`
- 标签始终可见，位于输入框上方，禁止用placeholder代替标签

### 弹窗
- 圆角16px，白色背景，弹窗阴影
- 蒙层：`rgba(0, 0, 0, 0.5)`半透明黑色
- 进入动画：从下往上滑入+淡入，250ms
- 关闭按钮位于右上角，点击蒙层或按ESC可关闭

### 表格
- 无边框设计，仅用底部`#F2F3F5`横线分隔行
- 表头背景`#F9FAFB`，文字灰色，字重600
- 行高56px，hover时背景`#F9FAFB`
- 操作按钮默认隐藏，hover时显示

### 图表
- 折线图：纯色线条，粗细2px，无渐变填充
- 饼图：低饱和色系，无复杂渐变
- 背景纯白，网格线`#F2F3F5`，极浅灰色
- tooltip极简设计，只显示核心数据

## 六、交互规范
### 微交互
- 所有可点击元素点击时缩小到0.98倍，松开恢复，过渡150ms
- 卡片hover时上移1px，阴影加深，过渡200ms
- 列表行hover时背景变浅灰，过渡150ms
- 页面切换淡入动画，250ms
- 表单提交时按钮显示loading状态，禁止重复点击

### 操作反馈
- 成功操作：绿色Toast提示，2秒自动消失
- 失败操作：红色Toast提示，直到用户关闭或点击重试
- 危险操作：二次确认弹窗，防止误操作
- 加载状态：骨架屏或加载动画，禁止空白等待

### 流畅度要求
- 所有页面加载时间<1s
- 列表滚动60fps，无卡顿
- 操作响应时间<100ms
- 大数据量时使用虚拟滚动，避免性能问题

---

## 七、无障碍规范
- 所有图标按钮必须添加`aria-label`属性
- 颜色不能是唯一的信息传递方式，必须搭配图标或文字
- 支持键盘导航，Tab键顺序符合视觉顺序
- 支持系统字体大小缩放，布局不破裂
- 支持`prefers-reduced-motion`，用户关闭动画时禁用所有非必要动画
- 图片必须添加`alt`属性说明

## 九、代码实现规则
1. 所有样式必须使用Tailwind CSS，禁止写原生CSS（全局变量除外）
2. 组件必须独立、可复用，禁止业务逻辑和UI耦合
3. 动画使用CSS transitions/transforms，禁止JS动画
4. 响应式设计必须覆盖移动端、平板、桌面端
5. 所有自定义颜色必须通过CSS变量引用，禁止硬编码色值
6. 严格遵循代码规范，保持代码风格统一

本设计系统为唯一UI开发标准，所有功能开发必须严格遵守，如有特殊需求需要修改，必须先更新本文件。
